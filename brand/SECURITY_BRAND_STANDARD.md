# AuthePay Security Brand Standard

## 1. Purpose

This document defines how AuthePay communicates security, trust, verification, and compliance across all customer-facing platforms.

Applies to:

- Website
- Mobile application
- Payment receipts
- Merchant dashboard
- Customer dashboard
- Emails
- API documentation
- Verification pages
- PDF documents


# 2. Brand Identity

Brand:
AuthePay

Company:
Braincade Holdings (Pty) Ltd T/A AuthePay Tech

Security Position:

"Trusted Digital Payments Infrastructure"


# 3. Official Logo Usage

The official AuthePay logo must be used across all security-related interfaces.

Logo location:

assets/logo.svg


## Logo Requirements:

- Maintain original proportions
- Do not stretch or distort
- Do not change colours
- Do not add unapproved effects
- Maintain clear space around logo
- Use high-resolution SVG whenever possible


## Required Logo Placement:

Website:
- Top navigation
- Security pages
- Footer

Mobile App:
- Splash screen
- Login screen
- Security verification screens

Documents:
- Receipts
- Invoices
- Statements
- Settlement reports

Emails:
- Header
- Footer where applicable


# 4. Security Visual Language

Security elements must communicate:

- Trust
- Protection
- Transparency
- Verification


Approved security indicators:

✓ Verified by AuthePay

✓ Secure Transaction

✓ Protected Payment

✓ Identity Verified

✓ Document Authenticity Confirmed


# 5. Trust Badge Standard

Every verified transaction may display:

AUTHEPAY VERIFIED

Includes:

- AuthePay logo
- Verification ID
- Timestamp
- QR verification code


# 6. Fraud Warning Design

Security warnings must be clear and professional.

Example:

"AuthePay will never request your password, PIN, or private security information."


# 7. Verification Page Standard

Every verification page must include:

- AuthePay logo
- Verification status
- Document/transaction ID
- Date and timestamp
- Security information


Example:

AUTHEPAY VERIFICATION

Status:
✓ Authentic

Reference:
AP-XXXXXXXX


# 8. API Security Branding

API documentation must include:

- AuthePay logo
- Security guidelines
- Authentication requirements
- Developer warnings


# 9. Compliance Presentation

Security and compliance sections must reference:

- Data protection principles
- Audit logging
- Fraud monitoring
- Transaction security


# 10. KaTSo AI OS Integration

KaTSo AI OS security monitoring may provide:

- Fraud risk alerts
- Suspicious activity detection
- Security recommendations
- Compliance monitoring


# 11. Developer Rules

Developers must:

- Use official AuthePay assets
- Follow approved colours
- Use approved icons
- Maintain security messaging consistency
- Never create unofficial security badges
