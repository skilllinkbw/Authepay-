# AuthePay Color System

## Purpose

This document defines the official AuthePay color system.

Every application, website, mobile app, dashboard, email, receipt, invoice, and marketing asset must use these colors consistently.

---

# Primary Brand Colors

Primary Blue

HEX: #2563EB

Purpose:
- Primary buttons
- Links
- Navigation
- Active elements
- Highlights

---

Secondary Black

HEX: #111827

Purpose:
- Headings
- Logo text
- Navigation
- Main text

---

Background White

HEX: #FFFFFF

Purpose:
- Page background
- Cards
- Forms
- Tables

---

# Functional Colors

Success

HEX: #16A34A

Use for:
- Successful payments
- Completed transactions
- Approved verification

---

Warning

HEX: #F59E0B

Use for:
- Pending transactions
- Verification in progress
- Alerts

---

Danger

HEX: #DC2626

Use for:
- Failed payments
- Fraud alerts
- Errors
- Security warnings

---

Information

HEX: #0EA5E9

Use for:
- Notifications
- Updates
- System messages

---

Neutral Gray

HEX: #6B7280

Use for:
- Secondary text
- Borders
- Placeholders
- Disabled components

---

# Dark Mode

Background

HEX: #0F172A

Cards

HEX: #1E293B

Text

HEX: #F8FAFC

---

# Accessibility

All color combinations must meet WCAG AA contrast standards.

Never rely on color alone to communicate important information.

---

# Rule

Only approved brand colors may be used throughout the AuthePay ecosystem unless explicitly authorized by the design team.
